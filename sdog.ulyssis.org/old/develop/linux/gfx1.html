  • Svgalib: This is a low level graphics library for Linux, based on VGAlib 1.2 by Tommy Frandsen. VGAlib supported a number of standard VGA graphics modes, as well as Tseng ET4000 high resolution 256-color modes. It also provides ET4000 hicolor/truecolor DAC support. There's also an ATI mach32 driver by Michael Weller.

    When the library is first used by a program at run-time, the chipset is detected and the appropriate driver is used. This means that a graphics program will work on any card that is supported by svgalib, if the mode it uses is supported by the chipset driver for that card. The library is upwardly compatible with VGAlib.

    Getting started with svgalib is rather easy, the best thing you can do is read the instructions that come with the src. Mabye somebody should write a faq someday ;-)

  • X: Probably by Using XLib
  • GGI: This is probably the future, some of you hardware freaks might even be interrested in joining the project.
    The Linux-GGI project wants to set up a General Graphics Interface for Linux that will allow easy use of graphic hardware and input facilities under the Linux OS. One might argue that there are already existing solutions and standards like X or OpenGL to deal with graphic issues, but the current implementations under Linux have several (sometimes serious) drawbacks. GGI does not want to substitute these existing standards nor does it want to implement these services completely inside the kernel.
