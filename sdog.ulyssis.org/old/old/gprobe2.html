Proba Generala II

28-29 january 1995

INTRODUCTION

Adrar Design is proud to invite all PC users to the party called Proba Generalna II, which will take place in Ostrowiec Swietokrzyski, Poland, on 28 of January, 1996. This is second edition of this party. Last edition was the first polish PC party ever held and there were about 130 sceners from all corners of Poland. Now we expect about 300 or more attenders from Poland and neighboring countries such as Germany or Slovakia. We do hope you will come, too.

SOME TECHNICAL STUFF

  • party will begin on 28.01.96 about 10.00am and it will end at the same time the next day - 24h of non-stop fun and compos!
  • entrance fee will be 20 new polish zlotys ($8). All the founds will be used for prizes and organizing costs.
  • attenders can bring their own machines to the party-place. You'd better bring long cable to connect to the current source.
  • organizers are not responsible for hardware damage during the party, or any other things like money or clothes.
  • if somebody would want to sleep please take something to do it i.e. sleeping bag or so.

HOW TO GET TO THE PARTY PLACE

Ostrowiec Swietorzyski is a small town lying in south-eastern Poland. You'd better take a train to the capital of Poland - Warsaw and then there is direct train connection to Ostrowiec. There are three trains: 7.05, 16.05, 21.05 which are directly going to Ostrowiec. On the railway station in Ostrowiec you will find some sheets with the info how to get to the party place.

COMPETITIONS

Compo machine will be IBM PC 486SX 50Mhz (108 points in SYSINFO) 4Mb of RAM, standard VGA, SB and GUS 1MB, operating system MS-DOS 5.0 or higher, memory manager HIMEM or EMM386.

Following suggestions, one group can compete with more than one production but the creators of demo couldn't be the same.

Adrar Design will take part in competitions. Before deadline all attenders will be able to test their demo on compo machine. Demos will be shown in random order.

These are the competitions:

  1. Demo compo Rules:
    • only 100% VGA compatible registers and modes
    • music support: SB and GUS. If demo will not have both devices, it will be disqualified. Environment settings BLASTER and ULTRASND will be set.
    • demo must run automatically, might have setup screen.
    • maximum size is 4096KB and maximal running time is 8 minutes. If one of these conditions will not be passed, demo will be disqualified.
  2. 64KB Intro compo (konkurs kr˘tkich demonstracji) Rules:
    • only 100% VGA compatible registers and modes
    • music support: SB and GUS. Environment settings BLASTER and ULTRASND will be set.
    • intro must run automatically, might have setup screen.
    • maximum size is 64KB and maximal running time is 5 minutes. If one of these conditions will not be passed, intro will be disqualified.
  3. 4KB Intro compo Rules:
    • only 100% VGA compatible registers and modes
    • no music is allowed. It's coders' competition.
    • intro must run automatically.
    • maximum size is 4096 bytes and maximal running time is 3 minutes. If one of these conditions will not be passed, intro will be disqualified.
    • compo version of intro can not display coder's name during compo. As intro nameshouldn't be used a name which can suggest who is the coder (i.e. Stoned by Stone/Dust).
  4. 4-Channel Music compo Rules:
    • 4 channels or less are allowed
    • music format playable by: DMP 4.0, CP 1.5, or trackers ST3 and FT2.
    • maxinum playing time 3.5 minutes. If song will be longer it will fade out.
    • author's name will be not displayed during presentation
    • one song from one musician
  5. Multichannel Music compo Rules:
    • 2-32 channels are allowed
    • music format playable by: DMP 4.0, CP 1.5, or trackers ST3 and FT2.
    • maxinum playing time 3.5 minutes. If song will be longer it will fade out.
    • author's name will be not displayed during presentation
    • one song from one musician
  6. Gfx compo Rules:
    • maximum size is 640x480x256
    • graphic format readable by QPV, CSHOW, VPIC
    • graphician should bring atleast three phases of making picture, one final result will be displayed.
    • using scanning or raytracing techniques will disqualify picture
    • compo version of picture cannot be signed by author.
    • one gfx from one artist
  7. Raytracing compo Rules:
    • maximum size is 640x480x16mln.
    • graphic format readable by QPV, CSHOW, VPIC
    • graphician should bring atleast three phases of making picture, one final result will be displayed.
    • using manual retouch techniques will disqualify picture
    • compo version of picture cannot be signed by author.
    • one ray from one artist
