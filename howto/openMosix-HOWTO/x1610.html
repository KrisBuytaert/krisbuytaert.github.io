Q: Where do I get openMosix?:

A: The RPMs and source for openMosix are available from our Downloads/Files Section. Please read the release notes first!

Also, Gentoo Linux's emerge sys-apps/openmosix-user and Debian GNU/Linux openMosix packages are available.

Q: Can I mix MOSIX and openMosix nodes in the same cluster?:

A: No. Just like the older MOSIX, you should not mix nodes because the protocols are subject to unannounced changes from version to version. In addition, every new version has bug fixes which warrant updating to the new kernels.

Q: How do I build openMosix?:

A:

  1. Start by unpacking both the Linux kernel sources and the corresponding openMosix distribution in a directory, say /usr/src.

  2. Then 

    $ cd /usr/src; tar xzf linux-2.x.xx.tar.gz ;gunzip openMosix2.x.xx.gz

  3. Apply the openMosix patches to the pristine Linux kernel sources with 

          $ patch -p1 openMosix2.x.xx-x
    The directory /usr/src/linux-2.x.xx now contains the 2.x.xx kernel sources with the openMosix patches. Compile and install the resulting kernel as usual.

Q: What are userland tools?:

A: Userland tools are a collection of administrative tools used to examine and control an openMosix node.
